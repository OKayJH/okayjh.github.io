# 3D展厅与模型分组

## 摘要
本文记录 3D 展厅的模型分组结构、模型文件入口、维护方法和本次 `Camera` / `RK3566` / `TestingTerminal` / `Car` / `Robot` 分组调整的实现细节。后续若继续增加新的 3D 模型，优先从 `src/pages/works.astro` 的 `modelGroups` 常量和 `public/models/` 目录入手，不需要重新全仓搜索。

## 功能概述
3D 展厅位于 `/works`，左侧显示分组后的模型列表，右侧通过 `model-viewer` 展示当前模型，并在下方浮层显示模型标题与说明。当前展厅分组包括：
- `Camera`
- `RK3566`
- `TestingTerminal`
- `Car`
- `Robot`
- `Toy`

## 背景与目标
- 原有展厅只包含 `PCB` 与 `Toy`，无法承载网关、检测终端、小车控制板和机器人控制板模型。
- 本次将原 `PCB` 分组中的相机板模型改归到 `Camera`。
- 本次新增 `RK3566`、`TestingTerminal`、`CarCtrl`、`CarBottom`、`RobCtrl`、`RobBottom` 六个板卡模型入口。
- 用户要求分组名和模型名使用英文，同时 `RK3566` 与 `TestingTerminal` 采用各自名称单独分组，所有板卡类标签统一使用 `PCB`，相关说明使用中文。

## 功能边界
| 项目 | 范围 |
| --- | --- |
| 包含 | 展厅模型分组、模型名称、模型路径、模型说明、板卡标签 |
| 不包含 | `model-viewer` 交互逻辑重构、模型文件内容修改、模型压缩优化 |

## 架构与模块关系
1. `src/pages/works.astro` 定义 `modelGroups`，负责左侧模型列表与默认模型。
2. `public/models/` 存放 `.glb` 模型文件，页面通过 `src` 路径直接引用。
3. 页面内 `setupShowroom()` 监听按钮点击，切换 `model-viewer` 的 `src`、标题与描述。
4. `src/layouts/Layout.astro` 提供公共站点壳层，但不参与展厅模型分组逻辑。

## 调用关系
1. 用户进入 `/works` 后，页面先读取 `modelGroups` 渲染左侧按钮列表。
2. `initialModel` 取 `modelGroups[0].models[0]`，作为右侧初始展示模型。
3. 点击左侧任意 `.model-btn` 后，`setupShowroom()` 更新查看器 `src` 和下方说明卡片。

## 运行与调试
| 场景 | 最小排查路径 |
| --- | --- |
| 新模型没有出现在左侧列表 | 先检查 `src/pages/works.astro` 的 `modelGroups` 是否加入数据 |
| 点击模型后不加载 | 检查 `public/models/` 文件名与 `src` 路径大小写是否一致 |
| 分组顺序不对 | 调整 `modelGroups` 数组顺序 |
| 模型标签不符合要求 | 直接修改 `tag` 字段，当前板卡类统一应为 `PCB` |
| 板卡说明不是中文 | 直接修改 `desc` 字段 |
