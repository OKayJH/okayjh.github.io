# 2026-03-12 回到顶部按钮 Plan

## 摘要
本次需求是在博客全站右下角增加一个“回到顶部”按钮。页面位于顶端时按钮隐藏；当页面从顶端向下滚动后按钮出现；点击后平滑返回页面顶部。实现将复用现有全站布局 `src/layouts/Layout.astro`，并新增独立组件承载按钮、滚动监听与样式，避免影响现有页面内容结构。

## 需求背景
- 用户希望博客界面右下角增加一个与示例相近的回到顶部按钮。
- 该按钮需要具备明确的显示规则：
  - 页面在最顶端时不显示。
  - 页面只要向下滚动离开顶端，就显示。
  - 点击后无论当前页面滚动到哪里，都可以返回顶部。
- 项目当前已经存在底部音乐播放器和右下角 AI 助手等浮动元素，新增按钮时需要避免遮挡和交互冲突。

## 目标与边界
| 项目 | 内容 |
| --- | --- |
| 目标 | 新增全站回到顶部按钮，支持滚动显示/隐藏和平滑返回顶部 |
| 边界 | 不修改文章列表逻辑、不改页面内容结构、不重构现有 AI 助手和音乐播放器 |
| 风格要求 | 最小改动、保持现有站点视觉风格、尽量避免与现有浮层重叠 |

## 现状分析
| 现象 | 文件路径 | 结论 |
| --- | --- | --- |
| 全站公共壳层由布局文件统一装载 | `src/layouts/Layout.astro` | 回到顶部按钮应挂在布局层，确保全站生效 |
| 页面已存在右下角浮动元素 | `src/components/AIAssistant.astro` `src/components/MusicPlayer.astro` | 新按钮需要避让 AI 助手与底部播放器 |
| 全局已开启平滑滚动 | `src/styles/global.css` | 点击按钮时仍可显式调用 `window.scrollTo({ behavior: 'smooth' })` |
| 项目已存在组件内联脚本挂载模式 | `src/components/FloatingClock.astro` `src/components/AIAssistant.astro` | 可沿用 `DOMContentLoaded` + `astro:page-load` 的挂载方式 |

## 方案设计
1. 新增 `src/components/BackToTopButton.astro`，封装按钮 DOM、滚动显示逻辑与局部样式。
2. 在 `src/layouts/Layout.astro` 中引入并挂载该组件，使其对首页、文章页、展厅页等统一生效。
3. 通过监听 `window.scroll` 判断 `window.scrollY > 0` 时显示按钮，否则隐藏。
4. 按钮点击后调用 `window.scrollTo({ top: 0, behavior: 'smooth' })` 返回顶部。
5. 若存在 `小O` 助手，则根据 `#ai-toggle-btn` 与 `#ai-panel` 的实际位置动态调整按钮：收起时放到左侧，展开时移到面板上方。
6. 完成后补齐专题文档、总目录、架构文档和常见问题文档。

| 风险点 | 影响 | 应对方式 |

| 与 `小O` 助手位置冲突 | 影响右下角交互 | 根据助手按钮和面板的实际位置动态避让 |
