# 开发环境与构建部署

## 摘要
本项目当前运行在 Windows 宿主机上，使用 PowerShell 作为终端，站点框架为 Astro。开发时可通过 Node 环境本地运行；生产侧目标是静态站点部署，但当前仓库包含 API 路由，生产构建仍依赖适配器配置。

## 环境区分
| 环境类别 | 当前信息 | 用途 | 备注 |
| --- | --- | --- | --- |
| 宿主机环境 | Windows 10 / PowerShell | 本地编辑、安装依赖、运行命令 | 本轮对话实际执行环境 |
| 开发运行环境 | Node.js + Astro | 本地开发、构建、预览 | `package.json` 提供 `dev/build/preview` 脚本 |
| 目标运行环境 | 浏览器 + 静态托管站点 | 访问博客内容 | `astro.config.mjs` 中 `site` 为 `https://okayblog.me` |
| 板端/嵌入式目标环境 | 不适用 | 无 | 本项目不是板端固件或驱动工程 |

## 构建与运行命令
| 命令 | 执行位置 | 用途 | 示例 | 风险 |
| --- | --- | --- | --- | --- |
| `npm install` | 宿主机项目根目录 | 安装依赖 | `npm install` | 会更新 `node_modules`，可能受网络环境影响 |
| `npm run dev` | 宿主机项目根目录 | 本地开发调试 | `npm run dev` | 长驻进程，重复开启前应确认已有实例 |
| `npm run build` | 宿主机项目根目录 | 生产构建验证 | `npm run build` | 当前仓库会因缺少生产适配器而失败 |
| `npm run preview` | 宿主机项目根目录 | 预览构建结果 | `npm run preview` | 依赖先完成构建 |

## 关键配置说明
| 文件路径 | 配置项 | 作用 | 备注 |
| --- | --- | --- | --- |
| `package.json` | `scripts.dev` | 启动 Astro 开发服务 | 命令为 `astro dev` |
| `package.json` | `scripts.build` | 触发生产构建 | 命令为 `astro build` |
| `astro.config.mjs` | `site` | 站点正式地址 | 当前为 `https://okayblog.me` |
| `astro.config.mjs` | `adapter` 条件配置 | 仅在开发模式启用 Node 适配器 | 生产侧未安装/启用适配器 |

## 最小部署说明
1. 先在宿主机执行 `npm install` 确保依赖完整。
2. 本地联调使用 `npm run dev`。
3. 生产构建前需先确认 API 路由策略与适配器配置，否则 `npm run build` 会失败。
4. 若目标仍为纯静态托管，需要确认哪些 API 路由应保留、迁移或关闭。

## 已知问题
| 问题 | 现象 | 结论 | 建议 |
| --- | --- | --- | --- |
| 生产构建缺少适配器 | `npm run build` 报 `NoAdapterInstalled` | 属于仓库现有配置问题，不是本次首页文案修改引入 | 后续单独梳理 API 路由与生产部署方式 |
| `astro check` 依赖未安装 | 运行时提示安装 `@astrojs/check` | 当前仓库尚未配置该校验依赖 | 如需类型校验，可单独评估是否补充依赖 |
