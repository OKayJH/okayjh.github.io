# 调试记录与常见问题

## 摘要
本文件记录站点日常维护时最常见的定位入口，重点覆盖浏览器页签标题、首页作者介绍、标签和站点短描述的修改路径，便于后续最小范围改动。

## 常见问题表
| 问题 | 现象 | 关键文件 | 结论 | 建议 |
| --- | --- | --- | --- | --- |
| 浏览器页签标题在哪里改 | 浏览器标签页显示的站点名或页面名不符合预期 | `src/pages/index.astro` `src/layouts/Layout.astro` | 首页自定义标题由首页传入，通用后缀由布局统一处理 | 先改页面 `title`，再看布局里的标题规范化逻辑 |
| 浏览器左上角小图标在哪里改 | 浏览器标签页前面的站点小图标不符合预期 | `public/favicon-okay-logo.png` `src/layouts/Layout.astro` | 图标文件在 `public/`，布局层负责引用 | 优先替换图标文件并确认 `<link rel="icon">` 指向正确 |
| 展厅模型分组在哪里改 | 3D 展厅需要新增模型或调整分类 | `src/pages/works.astro` `public/models/` | 分组和文案在 `modelGroups`，模型文件在 `public/models/` | 先核对文件名大小写，再修改分组数据 |
| 回到顶部按钮在哪里改 | 页面右下角回顶按钮需要调整位置、样式或交互 | `src/components/BackToTopButton.astro` `src/layouts/Layout.astro` | 按钮逻辑在独立组件，布局层负责全站挂载 | 先看按钮是否已挂入布局，再检查滚动监听和 `updatePosition()` 的动态避让逻辑 |
| 默认站点描述在哪里改 | 页面未单独传 `description` 时，描述仍显示旧定位 | `src/layouts/Layout.astro` | `defaultDescription` 是全站默认描述入口 | 统一站点定位时优先改这里 |
| 首页作者介绍在哪里改 | 首页首屏文案、弹窗介绍和标签与个人定位不一致 | `src/components/AuthorProfile.astro` | Hero 文案、弹窗文案和标签都在同一组件维护 | 优先在此文件原地更新，不要散落到其他文件 |
| 页脚站点短说明在哪里改 | 页脚仍显示旧站点名称或旧标语 | `src/components/Footer.astro` | 页脚文案单独维护 | 站点整体定位变化时应同步更新 |
| 构建失败但文案改动无报错 | 运行 `npm run build` 报 `NoAdapterInstalled` | `astro.config.mjs` `package.json` | 属于仓库现有部署配置问题，不是首页文案修改引入 | 后续单独梳理 API 路由与生产适配器 |
