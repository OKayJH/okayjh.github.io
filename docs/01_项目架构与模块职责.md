# 项目架构与模块职责

## 摘要
`OKay_Blog` 是一个基于 Astro 构建的个人博客站点。项目主体由页面路由、统一布局、复用组件和 Markdown 文章内容组成；首页作者介绍与浏览器页签标题分别由 `src/components/AuthorProfile.astro` 和 `src/layouts/Layout.astro` 控制，3D 展厅入口由 `src/pages/works.astro` 维护。

## 架构概览
1. `src/pages` 负责路由入口和页面级数据准备。
2. `src/layouts/Layout.astro` 负责统一 `<head>`、页签标题、欢迎动画和全站壳层。
3. `src/components` 负责首页作者区、导航、页脚等复用 UI。
4. `src/content/blog` 存放文章正文，`src/content/config.ts` 负责内容集合定义。
5. `astro.config.mjs` 和 `package.json` 负责站点基础配置与构建命令。

## 调用关系
1. 浏览器访问 `/` 时先进入 `src/pages/index.astro`。
2. `index.astro` 将页面标题、描述传给 `src/layouts/Layout.astro`。
3. `Layout.astro` 规范化标题后输出到 `<title>`，并统一渲染页头、页脚和公共壳层。
4. `index.astro` 同时引入 `src/components/AuthorProfile.astro`，渲染首页作者介绍和标签。
5. 浏览器访问 `/works` 时进入 `src/pages/works.astro`，由 `modelGroups` 驱动左侧分组列表和右侧 3D 模型切换。

## 风险说明
- 当前全站多个页面仍沿用旧的 `title` 传参写法，但 `Layout.astro` 已做兼容清洗，避免继续显示 `OKay's Blog`。
- 项目包含 API 路由，生产构建是否成功取决于适配器配置；这与本次文案调整无直接关系。
