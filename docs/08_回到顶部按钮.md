# 回到顶部按钮

## 摘要
本文记录博客全站“回到顶部”按钮的实现位置、显隐规则、避让策略和维护方法。该按钮由布局层统一挂载，页面处于顶部时隐藏，只要页面向下滚动离开顶部就显示，点击后平滑返回页面顶端，并会根据 `小O` 助手的收起/展开状态动态调整位置。

## 功能概述
回到顶部按钮是全站公共浮动控件，固定在页面右下角，主要服务于文章页、首页、展厅页等长页面浏览场景。其行为规则如下：
- 页面在顶部时隐藏按钮。
- 页面滚动后显示按钮。
- 点击按钮时平滑回到顶部。
- 若 `小O` 助手收起，按钮移动到它左侧。
- 若 `小O` 助手面板展开，按钮移动到面板上方。

## 背景与目标
- 博客当前包含较长的文章列表页和内容页，向下浏览后回到顶部成本较高。
- 项目已经存在底部音乐播放器和右下角 AI 助手，新增按钮时需要兼顾视觉一致性和交互避让。
- 本次目标是在不改动页面正文结构的前提下，为全站补充一个简单、稳定、可维护的回顶入口。

| 项目 | 范围 |

## 架构与模块关系
1. `src/layouts/Layout.astro` 负责全站挂载 `BackToTopButton` 组件。
2. `src/components/BackToTopButton.astro` 提供按钮 DOM、滚动监听、点击回顶和局部样式。
3. `src/components/AIAssistant.astro` 作为现有右下角浮层，提供 `小O` 按钮和面板的定位参考，供回顶按钮动态避让。
4. `src/styles/global.css` 中的 `html { scroll-behavior: smooth; }` 与按钮点击时的 `window.scrollTo()` 共同保证平滑回顶体验。

## 调用关系
1. 页面通过 `Layout` 渲染时，同时挂载 `BackToTopButton`。
2. `BackToTopButton` 在 `DOMContentLoaded` 或 `astro:page-load` 后执行 `mountBackToTop()`。
3. `mountBackToTop()` 查询 `小O` 助手的切换按钮和面板节点，并建立位置观察。
4. `updatePosition()` 在页面加载、窗口尺寸变化和助手面板切换时重新计算按钮位置。
5. 页面滚动时，`updateVisibility()` 根据 `window.scrollY > 0` 切换按钮显示状态。
6. 点击按钮后，`scrollToTop()` 触发平滑滚动返回顶部。

## 依赖关系
| 依赖项 | 类型 | 用途 | 所在位置 | 使用入口 | 备注 |
| --- | --- | --- | --- | --- | --- |
| `window.scrollTo` | 浏览器 API | 将页面滚动回顶部 | 浏览器运行时 | 按钮点击事件 | 无第三方依赖 |
| `astro:page-load` | Astro 页面事件 | 支持切页后重新挂载监听 | 浏览器运行时 | 组件内联脚本 | 与现有组件模式一致 |
| `#ai-toggle-btn` / `#ai-panel` | DOM 选择器 | 提供 `小O` 按钮和面板的实时位置参考 | `src/components/AIAssistant.astro` | `updatePosition()` | 用于动态避让 |
| `MutationObserver` | 浏览器 API | 监听助手面板开关带来的布局变化 | 浏览器运行时 | `mountBackToTop()` | 避免写死偏移 |
