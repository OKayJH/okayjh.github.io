# 硬件平台与资源映射

## 摘要
本仓库是 Web 博客站点工程，不是驱动、BSP、内核或板端应用工程。本文件的目标不是描述 SoC、寄存器或外设资源，而是明确区分“宿主机开发环境”“浏览器运行环境”和“站点静态资源入口”，避免后续维护时把网页项目误当成嵌入式目标工程处理。

## 环境与资源边界
| 项目维度 | 当前状态 | 说明 | 备注 |
| --- | --- | --- | --- |
| 板卡/SoC | 不适用 | 本仓库不直接运行在板端，也不包含板级初始化逻辑 | 文章内容里提到的芯片仅属于博客正文素材 |
| 驱动节点/设备树 | 不适用 | 站点运行期不依赖 `/dev`、DTS、内核驱动 | 不要把博客文章主题与站点运行依赖混淆 |
| 宿主机 | Windows 10 + PowerShell | 用于开发、编辑、构建和本地调试 | 当前对话执行环境 |
| 目标运行环境 | 浏览器 | 用户最终访问博客页面的运行环境 | 标题、作者介绍、页脚都在浏览器渲染 |

## 维护建议
1. 若后续博客正文继续出现各类芯片、开发板和内核主题，不代表站点运行环境发生变化。
2. 若要修改浏览器页签图标或头像资源，应优先检查 `public/` 下的对应文件和组件引用。
3. 若要修改作者定位与站点介绍，应优先修改首页组件和布局层元信息，而不是文章正文。
