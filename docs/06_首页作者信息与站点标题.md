# 首页作者信息与站点标题

## 摘要
本文记录首页作者介绍、标签、浏览器页签标题、页签图标和默认站点描述的实现入口。后续若需要再次调整“我是谁”“博客定位是什么”“浏览器页签显示什么”，优先修改本文列出的相关文件，不需要重新全仓搜索。

## 背景与目标
- 首页此前的作者文案过度聚焦具体 SoC 和芯片型号，不符合当前个人长期定位。
- 首页标签以具体芯片为主，表达粒度偏窄。
- 首页浏览器页签标题过于通用，无法直接体现“这是一个什么博客”。

本次目标是统一为以下方向：
- 站点名称：`OKay的博客`
- 站点定位：专注智能硬件、嵌入式系统、电子产品设计
- 个人目标：立志成为一名专业的智能硬件工程师

## 功能边界
| 项目 | 范围 |
| --- | --- |
| 包含 | 首页页签标题、页签图标、默认站点描述、作者介绍、标签、页脚短说明 |
| 不包含 | 文章正文、评论系统、编辑器页、API 路由、部署配置修复 |

## 架构概览
1. `src/pages/index.astro` 负责给首页传入专用 `title` 和 `description`。
2. `src/layouts/Layout.astro` 负责生成最终 `<title>`，并提供全站默认描述。
3. `src/components/AuthorProfile.astro` 负责首页作者介绍、标签和“关于我”弹窗。
4. `src/components/Footer.astro` 负责页脚处的站点短说明。
5. `public/favicon-okay-logo.png` 提供浏览器页签左侧图标素材，`Layout.astro` 负责引用。

## 调用关系
1. 浏览器进入首页后，`src/pages/index.astro` 将首页标题和描述传给 `Layout.astro`。
2. `Layout.astro` 通过 `normalizedTitle` 和 `pageTitle` 统一输出 `<title>`，同时引用 `public/favicon-okay-logo.png` 作为页签图标。
3. `index.astro` 同时渲染 `AuthorProfile.astro`，展示首页首屏个人介绍和标签。
4. `Layout.astro` 在页面底部渲染 `Footer.astro`，保证页脚短说明与首页定位一致。

## 运行与调试
| 场景 | 最小排查路径 |
| --- | --- |
| 页签标题不对 | 先看 `src/pages/index.astro` 是否传了期望标题，再看 `src/layouts/Layout.astro` 的标题清洗逻辑 |
| 页签左侧图标不对 | 先看 `public/favicon-okay-logo.png` 是否为目标素材，再看 `src/layouts/Layout.astro` 的 `rel="icon"` 引用 |
| 默认站点描述不对 | 直接检查 `src/layouts/Layout.astro` 的 `defaultDescription` |
| 首页介绍或标签不对 | 直接检查 `src/components/AuthorProfile.astro` |
| 页脚定位不一致 | 检查 `src/components/Footer.astro` |
