# OKay's Blog 内容更新指南

这篇指南专门为你编写，帮助你快速掌握如何在此博客中添加各种多媒体资源与展示项。

## 🎧 1. 如何添加或更换背景音乐

博客底部的全局音乐播放器会自动播放项目中的音频文件。

### 准备音频文件
- 推荐使用 `.mp3` 或 `.wav` 格式。
- 文件名建议使用纯英文、数字，例如：`lofi-study.mp3`。

### 操作步骤
1. 将你的音频文件放入项目的静态资源目录：`public/music/` 中（如果没有该文件夹，请新建一个）。
2. 打开音乐播放器组件文件：`src/components/MusicPlayer.astro`。
3. 在文件开头的 JavaScript (Frontmatter) 区域，找到 `songs` 数组列表，仿照现有格式添加你的音乐：

```javascript
const songs = [
  {
    title: "Lofi Study",        // 音乐名称
    artist: "Unknown",          // 歌手/作者
    src: "/music/lofi-study.mp3" // 文件路径 (注意：/music 代表 public/music，不需要写 public)
  },
  // 在这里继续添加新的音乐 {}
];
```
4. 保存文件，音乐播放器的列表将自动更新。

## 🧊 2. 如何添加并展示 3D 模型

我们使用了 Google 的 `<model-viewer>` 来在 `3D 展厅 (/works)` 页面丝滑渲染三维硬件资产。

### 准备模型文件
- `model-viewer` 原生支持的最优格式为 **`.glb`** 或 **`.gltf`**。
- 如果你使用的是 Altium Designer、KiCAD 等 EDA 软件：
  1. 将 3D PCB 导出为 `.step` 文件。
  2. 使用免费开源的 [Blender](https://www.blender.org/) 导入 `.step` 文件。
  3. (可选) 给不同部件上材质（比如黑色的阻焊层绿油、银色的引脚）。
  4. 选中模型，在 Blender 菜单点击 `文件(File) -> 导出(Export) -> glTF 2.0 (.glb/.gltf)`。
  5. 压缩优化: 包含大量元器件的 PCB 模型可能很大，可以使用网页工具（如 glTF-Pipeline）对导出的 GLB 文件进行压缩，保证博客加载速度。

### 操作步骤
1. 将导出的 `.glb` 文件放入静态资源目录：`public/models/`。（比如新建 `public/models/my_board.glb`）
2. 打开 3D 展厅页面文件：`src/pages/works.astro`。
3. 找到代码中的 `<div class="model-list">` 列表区域，复制一个现有的 `button` 按钮。
4. 按如下方式修改 `button` 的属性：

```html
<button class="model-btn" 
        data-src="/models/my_board.glb"            <!-- 替换为你的模型路径 -->
        data-title="我的新定制开发板"                 <!-- 模型卡片大标题 -->
        data-desc="这是我设计的首款包含 NPU 的 AI 开发板，支持多种外设。"> <!-- 详细描述文字 -->
    
    <div class="model-icon">
        <span class="material-icons-round">memory</span> <!-- 侧边栏图标 (比如 memory/developer_board) -->
    </div>
    
    <div class="model-info">
        <span class="model-name">AI 开发板</span>      <!-- 侧边栏显示的短名称 -->
        <span class="model-tag">硬件设计</span>        <!-- 侧边栏的小标签文字 -->
    </div>
</button>
```
5. 保存文件后，访问展厅页面，你的模型就已经可以点击并且丝滑互动了！

## 📝 3. 如何编写与发布博客文章

博客文章采用了 Astro 的内容集合引擎（Content Collections）并支持本地直接编辑展示。

### 草稿与可见性机制
你在本地开发者环境（`npm run dev`）下能看到并修改**所有**文章（包括带“草稿”标签的未完成内容）。
当你部署到线上 (GitHub Pages 等) 时，标记为草稿的文章将被自动隐藏。

### 操作步骤：通过开发者控制台 (推荐)
1. 在本地启动博客：`npm run dev`
2. 打开首页，点击右上角的 **创作** 按钮。
3. 填入文章标题、标签、描述等。对于封面图片，点击上传区域可直接本地选图上传。
4. 使用 Markdown 编辑器编写正文。编写完毕后，点击正上方工具栏保存。
5. 保存后可选择关闭草稿状态正式发布。

### 操作步骤：直接编辑文件（适合习惯原生 MD 的极客）
1. 所有博客文章源码存放于：`src/content/blog/` 目录下。
2. 你可以手动新建一个 `.md` 文件，例如 `how-to-build.md`。
3. 在文件最顶部添加必要的 Frontmatter 元数据信息：
```markdown
---
title: "我的第一篇技术文章"
date: 2026-03-02
description: "这是一篇简短的介绍文章"
tags: ["Linux", "开发板"]
draft: false
# 封面图，可将图片放在 public/images/ 中，然后引用
# cover: "/images/post-cover.jpg"
---

这里是 Markdown 正文，支持代码高亮：
```
