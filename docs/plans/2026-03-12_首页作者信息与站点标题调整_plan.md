# 2026-03-12 首页作者信息与站点标题调整 Plan

## 摘要
本次改动目标是统一首页作者定位和浏览器页签文案：去掉过度聚焦具体 SoC/芯片型号的介绍，改为“智能硬件、嵌入式系统、电子产品设计”方向，并扩充首页标签展示。

## 需求背景
- 用户希望浏览器页签像参考博客一样，能够直接体现“某某的博客”和个人定位。
- 当前首页作者介绍与弹窗文案仍聚焦瑞芯微与 `RK3588`、`RK3576` 一类芯片，不符合当前个人定位。
- 首页标签数量偏少，且主要是具体芯片标签，不适合继续作为个人长期标签展示。

## 目标与边界
| 项目 | 内容 |
| --- | --- |
| 目标 | 调整首页浏览器标题、默认描述、作者介绍与首页标签 |
| 边界 | 仅改首页与站点壳层文案，不改文章正文内容，不改评论、编辑器和 API 路由逻辑 |
| 风格要求 | 最小改动、保持现有 Astro 结构不变 |

## 现状分析
| 现象 | 文件路径 | 结论 |
| --- | --- | --- |
| 首页页签标题是 `首页 | OKay's Blog` | `src/pages/index.astro` | 标题过于通用，且站点名称是英文旧写法 |
| 默认描述是 `嵌入式 Linux · AI · 硬件开发` | `src/layouts/Layout.astro` | 站点定位与用户当前需求不完全一致 |
| 作者区文案含“专注瑞芯微全系列芯片” | `src/components/AuthorProfile.astro` | 需要替换为更通用的个人定位 |
| 标签为 `RK3588` 等具体芯片 | `src/components/AuthorProfile.astro` | 需要改成能力/方向标签 |

## 方案设计
1. 在 `src/layouts/Layout.astro` 中增加站点名称和默认描述常量，统一页签标题后缀。
2. 在 `src/pages/index.astro` 中为首页单独传入更完整的标题和描述。
3. 在 `src/components/AuthorProfile.astro` 中同步修改 Hero 文案、弹窗介绍和标签。
4. 在 `src/components/Footer.astro` 中同步更新页脚短说明，避免站点内外表述不一致。
