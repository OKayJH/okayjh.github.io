# 2026-03-12 3D展厅模型分组调整 Plan

## 摘要
本次调整目标是在 3D 展厅中加入新增的 PCB 3D 模型，并按英文分组展示。当前已确认 `CarBottom.glb`、`CarCtrl.glb`、`RobBottom.glb`、`RobCtrl.glb`、`RK3566.glb`、`TestingTerminal.glb` 均已到位，且已根据最新要求将 `RK3566` 与 `TestingTerminal` 拆分为各自独立分组。

## 需求背景
- 用户新增了多个 PCB 3D 模型，希望在展厅页面直接浏览。
- 当前展厅只包含 `PCB` 与 `Toy` 两类，尚未纳入小车控制板、人形机器人控制板、网关与检测终端相关模型。
- 用户要求分组名称和模型名称使用英文，并把同类模型放在一起展示。
- 用户进一步要求：
  - 原 `PCB` 分组改名为 `Camera`
  - `RK3566` 与 `TestingTerminal` 单独使用各自名称作为分类
  - `Car` / `Robot` 分组保留
  - 所有板卡类模型的标签统一使用 `PCB`
  - 板卡类模型说明统一改为中文

## 目标与边界
| 项目 | 内容 |
| --- | --- |
| 目标 | 调整为 `Camera` / `RK3566` / `TestingTerminal` / `Car` / `Robot` / `Toy` 分组，并完成板卡类模型中文说明与统一标签 |
| 边界 | 仅修改展厅页面数据与相关文档，不改 3D 查看器交互逻辑，不处理模型文件本体 |
| 风格要求 | 最小改动，复用现有 `modelGroups` 数据结构 |

## 现状分析
| 现象 | 文件路径 | 结论 |
| --- | --- | --- |
| 展厅模型数据直接写在页面常量 `modelGroups` 中 | `src/pages/works.astro` | 新增模型只需扩展该数组即可 |
| 当前展厅分组已扩展为多组 | `src/pages/works.astro` | 需要进一步将共享 `PCB` 分组拆分为 `RK3566` 与 `TestingTerminal` 两个独立分组 |
| `public/models/` 已存在全部目标模型文件 | `public/models/` | 现可直接进入展厅接入实现 |
| 当前 `PCB` 分组中的 `IMX415`、`OS08A20` 已是相机板 | `src/pages/works.astro` | 应更名为 `Camera` 更符合实际含义 |

## 方案设计
1. 将原 `PCB` 分组改名为 `Camera`，保留 `IMX415`、`OS08A20`。
2. 将 `RK3566` 与 `TestingTerminal` 分别拆成独立分组，分组标题直接使用模型名。
3. 保留 `Car` 分组，放入 `CarCtrl` 与 `CarBottom`。
4. 保留 `Robot` 分组，放入 `RobCtrl` 与 `RobBottom`。
5. 将所有板卡类模型的 `tag` 统一调整为 `PCB`，并改为中文说明。
6. 实现后补齐正式专题文档，并同步更新相关计划文档。
